# Frugal.sh - Positioning, Landing Page Copy, and 2-Week Launch Plan

## 1) Positioning

### One-line
Frugal.sh is the API for turning reusable AI skills into production workflows with benchmarking, evals, and cost control built in.

### Who it is for
- Builders shipping AI features fast
- PMs validating intent/routing quality before release
- Growth and ops teams automating repeatable research workflows

### Core promise
Ship reliable skill-powered automation without guessing quality, cost, or intent performance.

### Messaging pillars
1. **Outcome-first workflows**: Use skills to produce business outputs, not generic model calls.
2. **Trust through evals**: Run benchmarks and validate intent quality before launch.
3. **Cost-aware by default**: Keep latency and spend visible and controlled.
4. **Launch-ready velocity**: Go from idea to production workflow quickly.

## 3) 2-week launch plan

## Week 1 - Foundation + Proof

### Day 1-2: Narrative + assets
- Finalize positioning and ICP (primary: builders/PMs shipping AI workflows)
- Produce a 60-second product demo
- Create benchmark proof screenshot set (quality + latency + cost)

### Day 3-4: Landing + conversion
- Publish landing page with clear CTA
- Add 3 use-case sections with sample output
- Add waitlist/onboarding form with source tracking

### Day 5: Content seeding
- Publish launch thread on X + LinkedIn
- Publish technical walkthrough post (how eval gates work)
- Prep Product Hunt assets (tagline, gallery, first comment)

### Day 6-7: Directory batch 1
Submit to high-leverage platforms:
- Product Hunt
- Hacker News (Show HN)
- Indie Hackers
- There’s An AI For That
- Toolify
- NextGen Tools
- Dang

Track each submission in launch directory tracker.

## Week 2 - Distribution + Feedback Loop

### Day 8-9: Outbound activation
- Send targeted outreach to 50 qualified builders/PMs
- Offer “first workflow setup” for design partners
- Collect top objections and activation blockers

### Day 10-11: Directory batch 2
Submit remaining long-tail directories from `launch-directories-frugal.md`.

### Day 12: Live proof event
- Run a live benchmark/eval walkthrough
- Show baseline vs improved intent model performance
- Turn recording into short clips

### Day 13: Conversion push
- Publish customer/problem-focused post with ROI framing
- Add FAQ/pricing clarity from objections

### Day 14: Review + iterate
- Review funnel metrics
- Rank channels by activation quality
- Commit week-3 plan on top 2 channels only
