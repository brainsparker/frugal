Tool calls are the new tokens.

For most agentic workloads, the tool bill exceeds the model bill. Frugal is the cost-arbitrage layer for those tool calls — search, extract, browse, code-exec, embeddings — routed to a free/local provider first, falling back to the cheapest paid only when needed. Works with any model. One signed Go binary. Your keys. No account.

The rack-rate gap

The cheapest tools for AI.

Tool prices haven't fallen the way model prices have. A Firecrawl scrape costs more than the gpt-4o-mini call that consumes it. A Tavily search costs 25× a Serper search and 100× a self-hosted SearXNG. Frugal routes each call to the leftmost column that clears the recipe's quality bar — you keep the gap.

Frugal's job: try the leftmost column that's configured; fall back rightward only when the result fails the recipe's quality bar. The "premium paid" column isn't wrong — those providers exist for a reason. They're just the wrong default for cost optimization. Frugal makes them opt-in, not opt-out.

The rule

$0 first. Paid as fallback.

For every tool call, Frugal walks configured providers in ascending cost order. Free / local providers ($0) are tried first. The loop stops as soon as one provider returns a result that clears the recipe's quality bar. Premium-paid providers only run when explicitly opted in.

Classify
Detect the task and the capabilities required — search, extract, browse, code-exec, embeddings, transcription, or a multi-step combination.
Route cheapest-first
For each capability, walk providers in cost order. SearXNG before Serper before Tavily. Trafilatura before Firecrawl. Local Playwright before Browserbase.
Audit
Every response prints the decision — recipe, step list, provider per step, actual cost paid. frugal route previews without paying.

Flow

One run, end to end

frugal run "summarize this repo"
       │
       ├─ classify task → recipe ID + quality tier
       ├─ load recipe steps from config/use_cases/<id>.yaml
       ├─ for each step:
       │     ├─ tool  → call MCP tool (loopback) → routed provider
       │     └─ chat  → router picks cheapest model that clears the threshold
       ├─ bind {task}, {step1.results}, … into subsequent step inputs
       └─ print result, cost, and recipe path

Same engine whether you type frugal run or your agent calls frugal__search via MCP. One routing decision, two surfaces.

Surfaces

MCP to ship it. CLI to try it.

Agent stacks — Claude Code, Cursor, Claude Desktop, custom hosts — talk to Frugal via MCP (Model Context Protocol). That's where Frugal lives in production. The CLI is for evaluation, scripting, and the install flow itself. Same routing engine underneath.

Quality dial

Three tiers. Pick per call.

--quality cost for extractions and lookups. --quality high for planners and novel code. Default balanced for everything else.

high

Top-tier steps only. Planners, complex reasoning, novel code.

frugal run --quality high

balanced default

Best cost-quality tradeoff. The right choice ~80% of the time.

frugal run --quality balanced

cost

Cheapest viable recipe path. Classification, extraction, summaries.

frugal run --quality cost

Roadmap

Component status

Each capability is its own router. Shipping today: hosted chat and routed search. Phase 2 (next): local providers — SearXNG, Playwright, Trafilatura, Firecrawl, Browserless — and the $0-first routing rule that needs them. Pricing synced from models.dev at startup; extend by editing ~/.frugal/config/models.yaml.

Shipping — routes live traffic. Phase 2 — next to ship. Planned — on the roadmap.

Search partial

Serper shipping via the frugal__search MCP tool. SearXNG + Brave free tier land in Phase 2 to enable $0-first routing.

Extract phase 2

Trafilatura, readability.js, Mercury (free / local); Firecrawl as premium-paid fallback. Schema reserved.

Browse phase 2

Local Playwright + Chromium; Browserless cheap-paid; Browserbase premium-paid. Schema reserved.

Hosted chat shipping (internal)

OpenAI, Anthropic, Google chat completions inside recipe chat steps. Exposed as frugal__chat MCP tool in Phase 3.

Local chat phase 3

Ollama / LM Studio drivers for the showcase audience running Kimi / Qwen / Llama locally.

Embeddings planned

nomic-embed-text / bge-large (local); text-embedding-3-small (cheap paid); 3-large / Voyage (premium).

Transcription planned

whisper.cpp local; Deepgram cheap paid; OpenAI Whisper / Speechmatics premium.

Code exec planned

Local Docker sandbox; E2B cheap paid; Modal premium. For math, data-shaping, verification.

Semantic cache planned

Local sqlite + similarity for repeated / near-duplicate calls. Zero-cost win.

Multi-step agent planned

frugal run --agent mode — a cheap-model loop using Frugal tools when one call isn't enough.

Get started

Install

One Go binary, ~10 MB. No SaaS. No account. BYOK — Frugal reads OPENAI_API_KEY, ANTHROPIC_API_KEY, GOOGLE_API_KEY, TAVILY_API_KEY, SERPER_API_KEY from your environment and forwards calls upstream. Releases are cosign-signed; the installer verifies the checksum before moving the binary into place. 

$ curl -fsSL https://frugal.sh/install | bash

From source

$ git clone https://github.com/brainsparker/frugal.git && cd frugal && make build

Add Frugal to your agent

$ frugal mcp install
# auto-detects Claude Desktop, Cursor, Claude Code.
# merges 'frugal' into each detected config. idempotent.
